# v0.2.0 — ITool 架构重构 & 运行时桥接

**发布日期**：2026-06-12

从单体 CommandFn 函数指针架构迁移至面向对象的 ITool 接口 + X-macro 注册体系，新增运行时桥接、调试器集成、ClassDB 文档查询等 149+ 工具。

### 重大变更

- **工具注册机制重写** —— 从 `CommandFn` 函数指针 + codegen 脚本迁移至 `ITool` 接口 + X-macro 自动注册
  - 移除 `tools/codegen.py` 及 `// @tool register` 注释机制
  - 每个工具现在是独立的 `.hpp` 文件，实现 `ITool` 接口（`name()`、`category()`、`execute_impl()` 等）
  - 通过 `register/*.hpp` X-macro 文件自动收集，无需 `.cpp` 文件
- **工具命名统一** —— 所有工具名从 `snake_case_with_underscores` 迁移至简洁 `snake_case`
  - `get_node_position` → 已移除（由通用 `get_node_property` 兜底）
  - `create_gdscript` → `write_gd_script`，`edit_gdscript` → `patch_gd_script` 等
  - 旧单体文件（`node.cpp`、`property.cpp`、`scene.cpp` 等）全部拆分为独立工具
- **分类体系重构** —— 从 16 个扁平命令组重构为两级分类树
  - `editor_tools/scene_tree`、`editor_tools/scripts`、`editor_tools/workspace`、`runtime_tools/bridge` 等
  - 自动发现 + `find_tool` 全文搜索引擎
- **YAML 数据库降级为文档用途** —— `node_props/db/*.yaml`（283）和 `node_resource/db/*.yaml`（419）不再参与构建/注册
  - 属性工具改为运行时通过 Godot ClassDB 动态查询
- **轮询机制变更** —— 从 `_on_process_frame` 信号改为 `_process()` 虚函数直接重写

### 新增

- **ITool 工具框架** —— `tool_base.cpp/.hpp` 提供完整类型验证、schema 生成、分类管理
  - `IToolAdapter` 桥接 GDScript SDK 工具到 ITool 体系
  - `HandlerRegistry` 双轨注册：ITool 主表 + CommandFn 旁路表
- **运行时桥接系统** —— 编辑器进程与游戏进程双向通信
  - `GameBridgeNode`（游戏进程内 TCP 服务端，端口 9601）
  - `RuntimeBridge`（编辑器侧 TCP 客户端）
  - 6 个桥接工具：场景树查询、属性读写、方法调用、截图、输入模拟
  - 自动感知游戏启停（`is_playing_scene()`）
- **运行时生命周期工具**（6）—— `run_project`、`run_current_scene`、`run_specific_scene`、`stop_project`、`pause_project`、`set_movie_maker`
- **调试器集成**（15 个工具）—— FPS、内存、性能监控、断点管理、步进调试、调用栈、局部变量
- **ClassDB 文档查询工具**（5）—— `get_class_list`、`get_inheritance_chain`、`get_property_doc`、`get_method_doc`、`get_enum_doc`
- **工作区工具**（5）—— 2D/3D/脚本/资产库工作区切换
- **控制台工具**（4）—— 输出获取、错误/警告过滤、清空
- **动画工具**（5）—— AnimationPlayer 创建、动画剪辑、轨道、关键帧
- **UI 控件工具**（4）—— 控件创建、StyleBox、布局预设、主题覆盖
- **着色器工具**（3）—— 创建、读取、预设应用
- **TileMap 工具**（3）—— 信息获取、单元格设置/擦除
- **导出工具**（2）—— 预设列表、项目导出
- **项目脚手架**（1）—— `create_project` 创建新项目
- **项目图可视化**（1）—— `get_project_graph` 依赖关系图
- **视口截图** —— `capture_viewport`、`capture_game_viewport`
- **信号管理**（4）—— 连接/断开信号、列出信号、获取连接
- **节点组**（4）—— 添加/移除组、查询组成员
- **资源操作**（6）—— 保存/加载/创建/复制/清除/查询资源
- **属性兜底工具**（2）—— 通用 `get_node_property`/`set_node_property` 覆盖所有节点属性
- **C++ YAML 测试引擎** —— `TestEngine` 在编辑器进程内执行 YAML 测试
- **构建优化** —— sccache/ccache 自动检测、Unity build、lld-link、ryml header-only YAML

### 重构

- **四层工具体系**（ADR-015）—— Layer 0 兜底 → Layer 1 专用 → Layer 2 编辑器 → Layer 3 文档
- **SDK 集成 IToolAdapter** —— GDScript SDK 工具通过适配器纳入 ITool 体系
- **工具类别描述系统** —— `category_description()` 自动美化分类标签
- **目录结构扁平化** —— `extensions/src/built_in/tools/` 按功能域分子目录
- **HTTP 服务器模块化** —— Origin 校验安全增强，防止重入（`save_scene` 崩溃修复）
- **构建系统** —— 移除 PCH、`build.py` 缓存自动检测 + SSL 降级重试

### 修复

- `save_scene` 崩溃：`EditorProgress::step()` → `Main::iteration()` → 重入 → 改用 `save_scene_as(path, false)` 跳过预览
- `delete_file`/`paste_node`/`run_scene` 语义错误修复
- HTTP Origin 校验漏洞修复
- `HashMap` range-for 内 `erase()` 迭代器失效 → UB 崩溃
- integer 类型验证兼容 float（`tool_base.cpp`）
- `add_node` 兼容 `name` 参数别名
- `handler_registry` 分类标签下划线处理

### 移除

- `tools/codegen.py` 及 `// @tool register` codegen 机制
- 旧单体工具文件（`cmd_node.cpp`、`cmd_scene.cpp`、`cmd_property.cpp` 等）
- `tools/tool_schemas.json`（工具 schema 现由 ITool 接口动态生成）
