# v0.2.2 — 非破坏编辑引擎、操作录制/回放、撤销/重做系统与 164 工具

**发布日期**：2026-06-27

重磅扩展版本：引入影子场景非破坏编辑引擎、操作录制与回放系统、自管理撤销/重做栈、Workflow 多步骤编排引擎、多实例运行时桥接服务器，以及 10 个 MCP 运行时资源和 9 个动态 Prompt。工具总数 153→164。

### 重大变更

- **`get_tool_detail` 移除** — 由 `get_tools(name)` 替代，后者直接返回工具完整 schema。
- **`generate_client_config` 移除** — 客户端配置生成内联到编辑器底部面板 UI，不再作为独立工具。
- **运行时桥接重构：`RuntimeBridge` → `RuntimeBridgeServer`** — 多实例 TCP 服务器模型，支持异步 pending 请求 + SSE 事件推送，取代旧单连接客户端模式。
- **桥接协议改为长度前缀帧** — 更可靠的 TCP 帧定界，取代旧 `\n` 定界。
- **`search_docs` 改为 ClassDB 运行时引擎** — 不再依赖编辑器 help 系统，直接通过 Godot ClassDB 搜索类名、方法、属性、信号和枚举。
- **`get_class_info` 改为 ClassDB 静态 API** — 不再实例化对象，直接通过 `ClassDB::class_get_method_list` 等静态方法查询。
- **移除旧停靠面板 `McpDock`** — 功能合并至 `McpConsole` 状态栏。

### 新增

- **影子场景非破坏编辑引擎**（`scene_diff/`模块，4 工具）— `stage_scene_change` 快照当前场景到影子缓冲区，`preview_change` 预览未提交差异，`apply_changes` 应用影子变更到实际场景，`discard_changes` 丢弃影子缓冲区。需在项目设置中启用 `godot_mcp/shadow_scene_enabled`。
- **场景差异对比** — `diff_scene_states` 工具比较两个 `.tscn` 文件的结构差异（新增/删除节点、属性变化等）。
- **操作录制与回放系统**（`replay/`模块，2 工具）— `record_operations` 开始/停止录制工具调用序列，`replay_operations` 按顺序回放录制操作。
- **撤销/重做系统**（3 元工具）— `undo`、`redo`、`get_undo_history`。自带 `UndoManager` 自管理栈（通过 `godot_mcp/undo_max_steps` 配置，默认 64，范围 16-512）。10 个核心工具已注入撤销支持。
- **Workflow 多步骤编排引擎** — `execute_workflow` 元工具，支持 YAML/JSON 定义多步骤流水线、变量替换 `${vars.*}`、步骤结果链式引用 `${steps.<id>.result.<path>}`。
- **`run_editor_script`** — 在编辑器上下文中执行 EditorScript (.gd) 文件，捕获 stdout，支持超时控制（可配 100-30000ms）。
- **`play_animation`** — 动画播放控制，支持 play、stop、pause、seek、get_status 五种操作，可配置 speed_scale、blend_time、from_end。
- **`list_game_instances`** — 查询所有已连接的游戏运行实例，返回 ID、连接时间戳、运行时长。
- **破坏性操作确认对话框** — `McpConfirmDialog` 弹窗拦截标记为 `is_destructive` 的工具调用，支持"允许本次"、"拒绝"和"始终允许本会话"。
- **10 个 MCP 运行时资源** — `godot://editor-info`（编辑器元信息）、`godot://console`（编辑器错误计数）、`godot://breakpoints`（脚本断点列表）、`godot://performance`（FPS/内存/节点数等性能指标）、`godot://filesystem`（文件系统扫描状态）、`godot://signals`（当前场景所有信号）、`godot://groups`（当前场景所有分组）、`godot://classes`（ClassDB 注册类列表）+ URI 模板 `godot://scene-node/{path}`、`godot://class/{name}`。
- **9 个动态 Prompt** — `create_scene`、`create_node`、`fix_error`（含实时编辑器错误计数）、`explain_node`（含 ClassDB 继承链与方法/属性/信号计数）、`code_review`、`debug_session`、`animate_node`、`shadow_edit`（影子场景工作流引导）、`export_project`。
- **HTTP 端口自动回退** — 自动尝试 10 个端口（`http_port` + 0~9），避让已占用端口，通过 `get_info` 返回实际端口。
- **项目设置注册系统** — `godot_mcp/http_port`、`http_host`、`bridge_port`、`log_dir`、`max_log_entries`、`shadow_scene_enabled`、`undo_max_steps` 自动注册并显示在项目设置中，支持 `set_as_basic` 标记。
- **tools-list-changed 通知** — `HandlerRegistry` 工具注册表变化时自动通过 SSE 推送 `notifications/tools_list_changed` 事件。

### 重构

- **Pipeline 引擎模块化** — `extensions/src/testing/` 迁移至 `extensions/src/pipeline/`，提取 `PipelineRunnerBase` 抽象基类，新增 `WorkflowRunner`/`WorkflowParser`（工作流）和 `TestRunner`/`TestParser`（测试）两条独立跑道。
- **MCP 处理器资源扩展** — `handle_resources_list/read` 从 2 个资源扩展到 10 个资源 + 2 URI 模板，代码量 +470 行。
- **Prompt provider 增强** — `fix_error` 读取编辑器实时错误计数；`explain_node` 从 ClassDB 动态构建继承链和 API 统计；新增 `debug_session`、`animate_node`、`shadow_edit`、`export_project` 四个 Prompt。
- **HandlerRegistry 增强** — 新增 `notify_tools_changed` 回调、`get_tools_by_group` 分组查询、`get_all_category_paths` 全路径枚举、频率衰减（每帧 *0.99）。
- **编译器选项加固** — MSVC/Clang-cl 添加 `/EHsc`（C++ 异常处理模型），clang-cl 同步。
- **底部面板重构** — 移除 `McpDock` 停靠面板组件，`McpConsole` 集成状态栏显示、客户端配置一键生成。
- **插件初始化简化** — `load_config` 移除 `any_nil` 逻辑，项目设置由 `register_project_settings()` 统一注册。

### 性能

- 桥接消息上限 64KB → 4MB（支持更大截图传输）
- `search_docs` 零分配：不再实例化 Godot 对象，仅 ClassDB 静态查询

### 修复

- `call_tool` 日志记录 tool_name 未展开为有效工具名（原记录为 "call_tool"）
- 新场景/场景切换后撤销操作丢失场景根节点引用
- 破坏性操作确认对话框置顶（`FLAG_ALWAYS_ON_TOP`）
- 旧 `RuntimeBridge` 析构顺序问题（服务端先于客户端析构时崩溃）

### 测试

- 新增 7 个 YAML 测试文件（25-31）：Phase 2 undo/redo、run_editor_script、screenshot/call_method、TileMap、Phase 3 workflow、Phase 4 shadow scene、Phase 4 replay。
- 测试编排器增强：Godot 启动重试（最多 3 次）、`/run-tests` 就绪检查。
- 测试重组：3D 和 C# 测试独立为分离场景，减少相互干扰。
- 全层测试基础设施增强：`test_runner.cpp` 新增文件系统快照验证。
