# v0.2.1 — MCP 2026-07-28 协议升级、架构优化与 153 工具

**发布日期**：2026-06-18

大规模架构优化版本：MCP 协议升级至 2026-07-28（无状态纯 POST）、~420 个 YAML 数据库文件替换为运行时 ClassDB 查询、工具注册宏从 6 参数简化为 2 参数、构建系统模块化、测试框架重构为流水线架构。

### 重大变更

- **MCP 协议升级：2025-03-26 → 2026-07-28** — 删除全部 session 管理（~200 行：`Session` 结构体、`create_session`、`validate_session`、`cleanup_expired_sessions`、`generate_uuid`）。移除 `GET /mcp` 和 `DELETE /mcp` 端点、`Mcp-Session-Id` 头、`initialize`/`initialized` 握手。改为纯 `POST + OPTIONS` 通信，SSE 事件内联在 POST 响应体中推送。
- **工具注册宏简化** — `GODOT_MCP_TOOL` 从 6 参数减至 2 参数（`cls`、`is_destructive_val`）。`name()`、`category()`、`brief()` 现为 `ITool` 接口的 `noexcept` 虚方法。全部工具 input schema 迁移至 `SchemaBuilder` 流式 API。
- **工作区/调试器/控制台工具合并** — 31 个工具精简至 13 个：`debugger_break`/`debugger_continue`/`debugger_step_*` 合并为 `debugger_control`；`get_fps`/`get_memory_usage`/`get_object_count`/`get_physics_stats`/`get_render_stats` 合并为 `get_performance_monitors`；`get_console_errors`/`get_console_warnings` 合并为 `get_console_output`；`set_workspace_*` 合并为带 `mode` 参数的 `set_workspace`。
- **插件工具合并** — `enable_plugin`/`disable_plugin` 合并为带布尔参数的 `set_plugin_enabled`。
- **脚本工具模板重构** — GDScript 和 C# 版本共享通用的 C++ 模板（`ScriptTool<Lang>`）消除了代码重复，但仍注册为独立工具（`read_gd_script`/`read_csharp_script`、`write_gd_script`/`write_csharp_script` 等）。

### 新增

- **3 个全新工具分类** — `editor_tools/audio`（3）：`create_audio_player`、`set_audio_stream`、`list_audio_buses`；`editor_tools/navigation`（3）：`create_navigation_region`、`create_navigation_agent`、`bake_navigation_mesh`；`editor_tools/3d_scene`（3）：`create_mesh_instance_3d`、`create_light_3d`、`set_world_environment`。
- **动画树工具**（5）— `create_animation_tree`、`get_animation_tree_info`、`add_animation_node`、`add_transition`、`set_transition_condition`。
- **导出工具**（2）— `validate_export_presets`、`get_export_platforms`。
- **输入映射工具**（3）— `add_input_action`、`remove_input_action`、`add_input_event_binding`。
- **着色器工具**（2）— `get_shader_uniforms`、`set_shader_uniform`。
- **运行时桥接** — `wait_for_bridge` 工具轮询等待桥接连接（可配超时）；全部桥接工具新增 `timeout_ms` 参数。
- **元工具** — `generate_client_config` 生成 VS Code、Cline、Claude、Roo Code 等 MCP 客户端配置。
- **`is_destructive` 安全标志** — 16 个工具标记为破坏性操作，客户端需确认。
- **CORS 动态校验** — HTTP 服务器可配置来源白名单。
- **Undo 支持** — 通过 `ScriptEditor::undo_redo()` 集成撤销/重做。

### 重构

- **MCP handler 拆分** — `mcp_handler.cpp`（-60%，~350 行）拆分为 `tool_executor.cpp`（~250 行）和 `prompt_provider.cpp`（~214 行）。
- **构建系统模块化** — `main.py`（288 行）3 子命令（build/package/test）；`scripts/` 下 11 个 Python 模块负责检测、CMake 封装、addon 配置、打包、测试。
- **CMake 模块化** — 根 `CMakeLists.txt` 从 130→32 行；提取 `compiler.cmake`、`platform.cmake`、`unity.cmake`、`cache.cmake`、`lto.cmake`、`pgo.cmake`、`deep-clean.cmake`；CMakePresets.json 提供 13 个预设组合。
- **UI 组件** — 新建 `extensions/src/ui/`，含 `McpDock`（右侧面板）、`McpLogger`（C++ 日志器）、`McpConsole`（编辑器输出控制台）。
- **SDK 层** — `McpToolRegistry` 注册为 Engine 单例（C# 通过 `Engine.GetSingleton` 访问）；`HandlerRegistry` 双重分发：ITool 主表 + SDK `CommandFn` 旁路表。
- **死代码清理** — `extensions/src/lsp/`（~212 行）、`SearchEngine`（~170 行）、`pch.hpp`、`logging.hpp`、`refactor_casts.py`、`collect_node_props.py`、`plans/` 和 `design/phases/` 中 3 个设计计划文档。
- **YAML 数据库移除** — `node_props/db/`（~283 个 YAML）、`node_resource/db/`（~419 个 YAML）、`tools/collect_node_props.py`（~261 行）。属性工具改为运行时通过 Godot ClassDB 动态查询。
- **测试框架流水线架构** — `test_engine.cpp` 减半（~400 行）；新增 Pipeline→Stage→Step 层级（`pipeline_parser.cpp`、`pipeline_runner.cpp`、`pipeline_context.cpp`、`pipeline_types.hpp`）；25 个 YAML 测试文件覆盖全部工具分类。

### 性能

- 桥接轮询间隔 50ms→5ms（-90% 响应延迟）
- 增量 JSON 解析 O(n²)→O(n)
- 缓冲区限制 64KB→1MB
- `walk_project_dir` 改用 `std::filesystem`（~10x 加速）
- CPack STRIP 启用（Release zip -80%）
- Unity batch 算法：`min(cpu, ceil(files/4))`
