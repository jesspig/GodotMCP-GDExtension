# v0.2.1 — MCP 2026-07-28 Protocol, Architecture Optimization & 153 Tools

**Release Date**: 2026-06-18

Major architecture optimization release: MCP protocol upgraded to 2026-07-28 (stateless POST-only), ~420 YAML database files replaced by runtime ClassDB queries, tool registration macro simplified from 6 to 2 parameters, build system modularized, and the test framework refactored to pipeline architecture.

### Breaking Changes

- **MCP protocol upgrade: 2025-03-26 → 2026-07-28** — removed all session management (~200 lines: `Session` struct, `create_session`, `validate_session`, `cleanup_expired_sessions`, `generate_uuid`). Removed `GET /mcp` and `DELETE /mcp` endpoints, `Mcp-Session-Id` header, and `initialize`/`initialized` handshake. Now pure `POST + OPTIONS` with SSE events inlined in POST responses.
- **Tool registration macro simplified** — `GODOT_MCP_TOOL` from 6 parameters to 2 (`cls`, `is_destructive_val`). `name()`, `category()`, `brief()` are now `noexcept` virtual methods on the `ITool` interface. All tool input schemas migrated to `SchemaBuilder` fluent API.
- **Workspace/debugger/console tools merged** — 31 tools reduced to 13: `debugger_break`/`debugger_continue`/`debugger_step_*` merged into `debugger_control`; `get_fps`/`get_memory_usage`/`get_object_count`/`get_physics_stats`/`get_render_stats` merged into `get_performance_monitors`; `get_console_errors`/`get_console_warnings` merged into `get_console_output`; `set_workspace_*` merged into `set_workspace` with `mode` parameter.
- **Plugin tools merged** — `enable_plugin`/`disable_plugin` combined into `set_plugin_enabled` with boolean parameter.
- **Script tools template refactoring** — GDScript and C# variants share a common C++ template (`ScriptTool<Lang>`) eliminating code duplication, but are still registered as separate tools (`read_gd_script`/`read_csharp_script`, `write_gd_script`/`write_csharp_script`, etc.).

### New Features

- **3 new tool categories** — `editor_tools/audio` (3): `create_audio_player`, `set_audio_stream`, `list_audio_buses`; `editor_tools/navigation` (3): `create_navigation_region`, `create_navigation_agent`, `bake_navigation_mesh`; `editor_tools/3d_scene` (3): `create_mesh_instance_3d`, `create_light_3d`, `set_world_environment`.
- **Animation tree tools** (5) — `create_animation_tree`, `get_animation_tree_info`, `add_animation_node`, `add_transition`, `set_transition_condition`.
- **Export tools** (2) — `validate_export_presets`, `get_export_platforms`.
- **Input map tools** (3) — `add_input_action`, `remove_input_action`, `add_input_event_binding`.
- **Shader tools** (2) — `get_shader_uniforms`, `set_shader_uniform`.
- **Runtime bridge** — `wait_for_bridge` tool for polling bridge connection with configurable timeout; all bridge tools now accept `timeout_ms` parameter.
- **Meta tools** — `generate_client_config` tool produces MCP client configs for VS Code, Cline, Claude, Roo Code, etc.
- **`is_destructive` safety flag** — 16 tools marked as destructive with client confirmation required.
- **CORS dynamic validation** — configurable origin allowlist for HTTP server.
- **Undo support** — `undo`/`redo` via `ScriptEditor::undo_redo()` integration.

### Refactoring

- **MCP handler split** — `mcp_handler.cpp` (-60%, ~350 lines) split into `tool_executor.cpp` (~250 lines) and `prompt_provider.cpp` (~214 lines).
- **Build system modularization** — `main.py` (288 lines) with 3 subcommands (build/package/test); 11 Python modules in `scripts/` for detection, CMake wrapping, addon config, archiving, testing.
- **CMake modularization** — root `CMakeLists.txt` from 130→32 lines; extracted `compiler.cmake`, `platform.cmake`, `unity.cmake`, `cache.cmake`, `lto.cmake`, `pgo.cmake`, `deep-clean.cmake`; CMakePresets.json with 13 preset combinations.
- **UI components** — new `extensions/src/ui/` with `McpDock` (right panel), `McpLogger` (C++ logger), `McpConsole` (editor output console).
- **SDK layer** — `McpToolRegistry` registered as Engine singleton (accessible from C# via `Engine.GetSingleton`); `HandlerRegistry` dual dispatch: ITool primary + SDK `CommandFn` sidetable.
- **Dead code removal** — `extensions/src/lsp/` (~212 lines), `SearchEngine` (~170 lines), `pch.hpp`, `logging.hpp`, `refactor_casts.py`, `collect_node_props.py`, 3 design plan docs in `plans/` and `design/phases/`.
- **YAML databases removed** — `node_props/db/` (~283 YAML files), `node_resource/db/` (~419 YAML files), `tools/collect_node_props.py` (~261 lines). Property tools now query Godot ClassDB at runtime.
- **Test framework pipeline architecture** — `test_engine.cpp` halved (~400 lines); new Pipeline→Stage→Step hierarchy (`pipeline_parser.cpp`, `pipeline_runner.cpp`, `pipeline_context.cpp`, `pipeline_types.hpp`); 25 YAML test files covering all tool categories.

### Performance

- Bridge poll interval 50ms→5ms (-90% response latency)
- Incremental JSON parsing O(n²)→O(n)
- Buffer limit 64KB→1MB
- `walk_project_dir` uses `std::filesystem` (~10x faster)
- CPack STRIP enabled (Release zip -80%)
- Unity batch algorithm: `min(cpu, ceil(files/4))`

### Fixes

- **6 P0 bugs** — CRLF injection (HTTP response splitting), TreeItem double-free (Tree destructor + `memdelete`), type validation bypass, `ctx.root` dangling pointer (after `save_scene`), register leak, SecuritySession leak
- `new_scene` crash on empty scene
- `HashMap` range-for `erase()` iterator invalidation → UB crash
- `TreeItem` double-free on Tree destructor
- MCP session cancel request key conflict
- Compiler warnings across Clang, GCC, MSVC
- `register_itools.cpp` unity build conflict (marked `SKIP_UNITY_BUILD`)
- ryml clang-cl `common.hpp` patch for `C4_MSVC` detection
- `args_get_typed` type safety for `Variant` conversions

### Removed

- **YAML databases** — `node_props/db/*.yaml` (~283), `node_resource/db/*.yaml` (~419)
- **LSP client** — `extensions/src/lsp/` (~212 lines, compiled but zero callers)
- **SearchEngine** — ~170 lines, replaced by HandlerRegistry search
- **Precompiled header** — `pch.hpp` (~39 lines)
- **Old logging header** — `logging.hpp` (~5 lines)
- **Legacy build scripts** — `build.py`, `tools/collect_node_props.py`, `refactor_casts.py`
- **13 redundant workspace tools** — merged into composite tools with mode parameters
- **Design docs** — `design/competitive-analysis.md`, `design/phases/*.md`, `design/v2-optimization-plan.md`, `plans/` directory (6 files)
- **Build system** — inline CMake options → modular `cmake/*.cmake`
