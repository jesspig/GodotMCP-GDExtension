# v0.1.5 — Pure C++ GDExtension Rewrite

**Release Date**: 2026-05-30

Removed Rust and Python components, fully migrated to C++ GDExtension.

### Breaking Changes

- **Removed Rust legacy code** — deleted all Rust crates (`crates/core`, `crates/gdext`, `crates/server`)
- **Removed Python MCP server** — deleted `server/` directory and its test suite
- **Removed planning docs** — deleted `docs/plan/` directory

### New Features

- **Pure C++ GDExtension implementation** — `extensions/gdext/` fully replaces the old Rust + Python architecture
  - HTTP server (`ipc/http_server.cpp`) — zero-dependency manual HTTP/1.1
  - MCP handler (`mcp/mcp_handler.cpp`) — JSON-RPC 2.0 + SSE
  - Handler registry (`commands/handler_registry.cpp`) — function pointer routing
- **Tool Schema system** — `tools/tool_schemas.json` (1977 lines), copied to output at build time
- **All 122 tools rewritten in C++** — covering scenes, nodes, properties, scripts, search, collision, editor control, project settings, input mapping, plugin management, Autoload
- **LSP client** — C++ version `lsp/client.cpp`, connects to Godot LSP via StreamPeerTCP
- **Utility functions** (`cmd_utils.cpp`) — `resolve_node`, `undoable_set`, JSON toolchain

### Refactoring

- **Single-process architecture** — removed dual-process (GDExtension + Python Server) architecture, all logic within GDExtension
- **Streamable HTTP transport** — replaced WebSocket transport
- **Streamlined build system** — pure CMake + `build.py` Python wrapper, removed Rust/Cargo toolchain
- **CI/CD update** — cross-platform CMake build (Ubuntu/MacOS/Windows), tag-and-release

### Fixes

- Multiple bugs in scene/node/script operations
- HTTP server robustness improvements
- Removed WebSocket server legacy code
