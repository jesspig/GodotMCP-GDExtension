# v0.2.2 — Non-destructive Editing, Operation Recording/Replay, Undo/Redo & 164 Tools

**Release Date**: 2026-06-27

Major expansion release: shadow scene non-destructive editing engine, operation recording and replay system, self-managed undo/redo stack, Workflow multi-step orchestration engine, multi-instance runtime bridge server, 10 MCP runtime resources, and 9 dynamic prompts. Tool count 153→164.

### Breaking Changes

- **`get_tool_detail` removed** — replaced by `get_tools(name)` which returns the full tool schema directly.
- **`generate_client_config` removed** — client config generation inlined into the editor bottom panel UI, no longer a standalone tool.
- **Runtime bridge refactored: `RuntimeBridge` → `RuntimeBridgeServer`** — multi-instance TCP server model supporting async pending requests + SSE event push, replacing the old single-connection client pattern.
- **Bridge protocol switched to length-prefixed framing** — more reliable TCP frame delimiting, replacing the old `\n`-delimited protocol.
- **`search_docs` switched to ClassDB runtime engine** — no longer relies on the editor help system; searches class names, methods, properties, signals, and enums directly through Godot ClassDB.
- **`get_class_info` switched to ClassDB static API** — no longer instantiates objects; queries metadata via `ClassDB::class_get_method_list` and related static methods.
- **Old `McpDock` panel removed** — functionality merged into the `McpConsole` status bar.

### New Features

- **Shadow scene non-destructive editing engine** (`scene_diff/` module, 4 tools) — `stage_scene_change` snapshots the current scene to a shadow buffer, `preview_change` previews uncommitted diffs, `apply_changes` applies shadow changes to the real scene, `discard_changes` discards the shadow buffer. Requires `godot_mcp/shadow_scene_enabled` in project settings.
- **Scene diff comparison** — `diff_scene_states` tool compares two `.tscn` files and returns structural differences (added/removed nodes, property changes, etc.).
- **Operation recording & replay system** (`replay/` module, 2 tools) — `record_operations` starts/stops recording tool call sequences, `replay_operations` replays recorded operations in sequence.
- **Undo/redo system** (3 meta tools) — `undo`, `redo`, `get_undo_history`. Self-managed `UndoManager` stack (configurable via `godot_mcp/undo_max_steps`, default 64, range 16-512). 10 core tools are undo-injected.
- **Workflow multi-step orchestration engine** — `execute_workflow` meta tool supporting YAML/JSON multi-step pipeline definitions, `${vars.*}` variable substitution, and `${steps.<id>.result.<path>}` step result chaining.
- **`run_editor_script`** — executes EditorScript (.gd) files in the editor context, captures stdout, supports configurable timeout (100-30000ms).
- **`play_animation`** — animation playback control supporting play, stop, pause, seek, and get_status actions with configurable speed_scale, blend_time, and from_end.
- **`list_game_instances`** — queries all connected game runtime instances, returning IDs, connection timestamps, and uptime.
- **Destructive operation confirmation dialog** — `McpConfirmDialog` popup intercepts tools marked as `is_destructive`, with "Allow Once", "Deny", and "Allow All for Session" options.
- **10 MCP runtime resources** — `godot://editor-info`, `godot://console` (editor error count), `godot://breakpoints` (script breakpoints), `godot://performance` (FPS/memory/node count), `godot://filesystem` (scan status), `godot://signals` (current scene signals), `godot://groups` (scene groups), `godot://classes` (ClassDB registered classes) + URI templates `godot://scene-node/{path}`, `godot://class/{name}`.
- **9 dynamic prompts** — `create_scene`, `create_node`, `fix_error` (with live editor error count), `explain_node` (with ClassDB inheritance chain and method/property/signal counts), `code_review`, `debug_session`, `animate_node`, `shadow_edit` (shadow scene workflow guidance), `export_project`.
- **HTTP port auto-fallback** — automatically tries 10 ports (`http_port` + 0~9), reporting the actual port via `get_info`.
- **Project settings registration** — `godot_mcp/http_port`, `http_host`, `bridge_port`, `log_dir`, `max_log_entries`, `shadow_scene_enabled`, `undo_max_steps` auto-registered and visible in project settings with `set_as_basic` markers.
- **Tools-list-changed notification** — `HandlerRegistry` automatically pushes `notifications/tools_list_changed` events via SSE when the tool registry changes.

### Refactoring

- **Pipeline engine modularization** — `extensions/src/testing/` migrated to `extensions/src/pipeline/`, `PipelineRunnerBase` abstract base extracted, two independent tracks added: `WorkflowRunner`/`WorkflowParser` (workflows) and `TestRunner`/`TestParser` (tests).
- **MCP handler resource expansion** — `handle_resources_list/read` expanded from 2 to 10 resources + 2 URI templates (+470 lines of code).
- **Prompt provider enhancements** — `fix_error` reads live editor error count; `explain_node` builds inheritance chain and API stats from ClassDB at runtime; 4 new prompts: `debug_session`, `animate_node`, `shadow_edit`, `export_project`.
- **HandlerRegistry enhancements** — new `notify_tools_changed` callback, `get_tools_by_group` grouped queries, `get_all_category_paths` full path enumeration, frequency decay (×0.99 per frame).
- **Compiler hardening** — `/EHsc` added for MSVC and clang-cl (C++ exception handling model).
- **Bottom panel refactoring** — `McpDock` panel removed; `McpConsole` integrates status bar display and one-click client config generation.
- **Plugin initialization simplified** — `load_config` removed `any_nil` logic; project settings now registered centrally via `register_project_settings()`.

### Performance

- Bridge message limit 64KB → 4MB (supports larger screenshots)
- `search_docs` zero-allocation: no longer instantiates Godot objects, only ClassDB static queries

### Fixes

- `call_tool` log recorded "call_tool" as the tool name instead of the effective tool name
- Undo operations missing scene root reference after scene creation/switch
- Confirmation dialog now always-on-top (`FLAG_ALWAYS_ON_TOP`)
- Old `RuntimeBridge` destruction order issue (server destroyed before client could disconnect)

### Tests

- 7 new YAML test files (25-31): Phase 2 undo/redo, run_editor_script, screenshot/call_method, TileMap, Phase 3 workflow, Phase 4 shadow scene, Phase 4 replay.
- Orchestrator enhancements: Godot startup retry (up to 3 attempts), `/run-tests` readiness check.
- Test reorganization: 3D and C# tests separated into isolated scenes to reduce interference.
- Test infrastructure: `test_runner.cpp` adds filesystem snapshot verification.
