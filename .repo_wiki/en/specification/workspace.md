# Cargo Workspace

## Structure

```toml
[workspace]
resolver = "2"
members = ["crates/core", "crates/server", "crates/gdext"]

[workspace.package]
version = "0.1.0"
edition = "2021"
```

Workspace members:

| Crate | Type | Description |
|-------|------|-------------|
| `crates/core` | lib | Shared protocol types |
| `crates/server` | bin | MCP server binary |
| `crates/gdext` | cdylib | Godot GDExtension |

## Pinned Dependencies

```toml
# crates/server/Cargo.toml
rmcp = "=1.7"          # strictly pinned
tokio = "1"
tokio-tungstenite = "0.18"
clap = { version = "4", features = ["derive"] }

# crates/gdext/Cargo.toml
godot = "=0.5"         # strictly pinned
tokio = "1"            # match server version
tokio-tungstenite = "0.18"
```

- **`godot = "=0.5"`**: strictly pinned — incompatible updates break builds
- **`rmcp = "=1.7"`**: strictly pinned — API breakage expected
- `Cargo.lock` IS committed (binary crate)

## Addon Manifest

`godot/addons/godot_mcp/` is the Godot Editor addon directory:

```
godot/addons/godot_mcp/
├── plugin.cfg          # Generated by CMake from Cargo.toml version
├── godot_mcp.gdextension  # GDExtension entry point
└── bin/
    ├── godot_mcp_gdext.dll   # Build artifact (.gitignore)
    └── .gitkeep
```

`plugin.cfg`:

```ini
[plugin]
name="Godot MCP"
description="MCP bridge for Godot Editor"
author="..."
version="0.1.0"  # auto-synced from Cargo.toml
script=""        # intentionally empty — all logic is native
```

`godot_mcp.gdextension`:

```ini
[configuration]
entry_symbol = "gdextension_init"

[libraries]
windows.debug.x86_64 = "res://addons/godot_mcp/bin/godot_mcp_gdext.dll"
windows.release.x86_64 = "res://addons/godot_mcp/bin/godot_mcp_gdext.dll"
```
