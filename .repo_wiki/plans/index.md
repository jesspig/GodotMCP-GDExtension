# GodotMCP 优化重构执行计划

> 基于 5 个并行分析 agent 的全面审计结果，设计了 **57 个任务节点、8 个执行批次** 的 DAG 并行执行方案。

## 核心指标

| 维度 | 数值 |
|------|------|
| 任务节点总数 | 57 |
| 执行批次 | 8（含 2 个验证关卡） |
| 最大并行度 | 10 个 subagent / 批次 |
| 预估总耗时 | ~4-5 小时（含编译验证） |
| 涉及文件 | ~80+ |
| 预估代码变化 | -3000 行（删减为主） |

## 执行原则

1. **文件域隔离** — 同批次并行 subagent 绝不触碰同一文件
2. **基础设施先行** — Batch 1 创建共享模板，后续批次才能使用
3. **每批一验** — 每个批次完成后必须编译通过 + 现有测试通过
4. **渐进式风险** — P0 bug 最先修复（Batch 0），高风险改动有独立验证
5. **可中断** — 任何批次失败不阻塞无关批次，仅阻塞下游依赖

## 快速启动

```
执行顺序：Batch 0 → Gate → Batch 1 → Gate → Batch 2 → Batch 3 → Gate → ... → Batch 7 → Final Gate

每批次流程：
1. 启动 N 个并行 subagent（N = 批次数，max 5）
2. 等待全部完成
3. 运行验证门禁（编译 + 测试）
4. 通过后进入下一批次
```
