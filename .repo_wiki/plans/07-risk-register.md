## 1. 风险评估矩阵

### 1.1 定量评分标准

| 概率 | 分值 | 影响 | 分值 |
|------|------|------|------|
| 极低 (<5%) | 1 | 极低 | 1 |
| 低 (5-20%) | 2 | 低 | 2 |
| 中 (20-50%) | 3 | 中 | 3 |
| 高 (50-80%) | 4 | 高 | 4 |
| 极高 (>80%) | 5 | 极高 | 5 |

**风险值 = 概率 × 影响**。阈值：≥12 红色，6-11 黄色，≤5 绿色。

## 4. 储备管理

### 4.1 储备计算

**应急储备**（基于 PERT 方差）：
```
关键路径: H → I → K → 测试
σ² = 0.25 + 0.44 + 0.11 + 0.25 = 1.05
σ = 1.02d

应急储备 = 2σ = 2.04d ≈ 2d
（95% 置信水平，即有 95% 概率不需动用管理储备）
```

**管理储备**（基于项目复杂度）：
```
管理储备 = 基础工期 × 15% = 12.84d × 15% = 1.93d
考虑未识别风险的不确定性，取整为 5d
```

### 4.2 储备分配计划

| Sprint | 基础工期 | 应急储备 | 管理储备 | 含储备工期 |
|--------|---------|---------|---------|----------|
| Sprint 1 | 3d | 0.5d | 1d | **4.5d** |
| Sprint 2 | 4d | 0.5d | 1.5d | **6d** |
| Sprint 3 | 9d | 1d | 2.5d | **12.5d** |
| **总计** | **16d** | **2d** | **5d** | **23d** |

### 4.3 储备使用规则

| 规则 | 说明 |
|------|------|
| **应急储备** | 由主 Agent 根据风险触发条件决定使用，记录使用原因 |
| **管理储备** | 需明确记录使用原因，仅用于未识别风险或范围变更 |
| **审批阈值** | 单次使用 >1d 需与用户确认 |
| **监控指标** | 储备使用 >50% 时触发项目状态审查 |
| **燃尽跟踪** | 每个 Sprint 结束时更新储备燃尽图 |
