## 1. 任务节点定义（含 PERT 三点估算）

### PERT 公式

```
TE = (O + 4M + P) / 6    期望时间
σ  = (P - O) / 6          标准差
σ² = ((P - O) / 6)²       方差
```

## 3. 拓扑排序与并行批次

### Batch 0 — 无依赖（可立即并行执行）

```
┌─────────────────────────────────────────────────┐
│ Batch 0: 4 个节点可完全并行                       │
│                                                  │
│  A (Q1)  ─── 1d                                  │
│  B (Q2)  ─── 3d                                  │
│  C (Q3)  ─── 2d                                  │
│  D (L3)  ─── 1.5d                                │
│  G (L1)  ─── 3d                                  │
│  H (WS-Codec) ─── 3d                             │
│                                                  │
│  关键路径: B (3d) + E (2d) + F (3d) = 8d         │
│  批次耗时: max(1, 3, 2, 1.5, 3, 3) = 3d          │
```

**Subagent 调度方案（Batch 0）**：

| Subagent | 分配节点 | 文件域 | 无冲突保证 |
|----------|---------|--------|-----------|
| SA-1 | A (Q1) | `editor_plugin`, `mcp_handler`, `handler_registry`, `mcp_dock` | 仅修改 ProjectSettings 读写 + Dock UI |
| SA-2 | B (Q2) | `pending_operation.hpp`（新）, `mcp_confirm_dialog`（新）, `mcp_handler`（增量） | 新文件 + mcp_handler 新增 pending 逻辑 |
| SA-3 | C (Q3) | `bridge.hpp/cpp`, `game_bridge.hpp/cpp` | 完全独立的运行时桥接文件 |
| SA-4 | D (L3) | `editor_plugin`（_enter_tree）, `mcp_dock`（端口警告） | 与 SA-1 的 mcp_dock 修改冲突 → 合并到 SA-1 |
| SA-5 | G (L1) | `.github/workflows/release.yml`（新）, `main.py` | 完全独立的 CI 文件 |
| SA-6 | H (WS-Codec) | `websocket.hpp/cpp`（新） | 完全独立的新文件 |

**冲突分析**：
- SA-1 (Q1) 和 SA-4 (L3) 都修改 `mcp_dock.hpp/cpp` → **合并为 SA-1+D**
- SA-2 (Q2) 修改 `mcp_handler.hpp/cpp`（新增 pending 逻辑）与 SA-1 (Q1) 修改同一文件 → **SA-1 先完成，SA-2 增量追加**
- 其余节点文件域完全隔离
